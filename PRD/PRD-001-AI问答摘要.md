## 1. 背景与业务问题

### 1.1 业务背景
- **当前业务阶段**：AI情报网站 1.0 已发布，用户可浏览新闻，但缺乏智能化交互能力
- **相关系统/模块**：新闻搜索、新闻详情、AI摘要

### 1.2 核心问题
- **现象描述**：用户获取信息效率低，需要逐条阅读新闻才能了解详情
- **根因分析**：现有产品仅提供新闻列表，缺乏智能化信息整合能力
- **若不解决的影响**：用户留存率低、访问深度不足、差异化竞争力弱

---

## 2. 目标与成功标准

### 2.1 业务目标
- 提升用户获取信息效率，实现"提问即答案"

### 2.2 产品目标
- 用户可通过自然语言提问，获取AI整合的相关新闻摘要

## 4. 方案设计与决策过程（重点）

### 4.1 方案对比

| 方案 | 描述 | 优点 | 缺点 |
|------|------|------|------|
| 方案 A：简单关键词匹配 | 根据用户问题提取关键词，匹配相关新闻标题 | 实现简单、速度快 | 回答质量低，无法整合 |
| 方案 B：RAG + LLM 摘要 | 构建新闻向量库，用LLM理解问题并生成摘要 | 回答精准、可整合多条信息 | 实现复杂、成本高 |
| 方案 C：外部API调用 | 调用第三方AI API（如Perplexity API） | 功能强大、体验好 | 依赖外部服务、有成本 |

### 4.2 最终方案选择
- **选择方案**：方案 B（RAG + LLM 摘要）
- **选择原因**：平衡成本与体验，可控性强，能真正提供价值
- **取舍说明**：
  - 相比方案A，方案B能提供真正的AI摘要价值
  - 相比方案C，方案B长期成本可控，不依赖外部服务

## 6. 功能与逻辑说明

### 6.1 功能说明

#### 功能名称：AI问答摘要

**功能描述**：用户输入问题，AI从新闻库中检索相关内容并生成摘要回答

**输入**：
- 用户问题（文字，≤500字符）
- 上下文（可选，上一轮问答内容）

**输出**：
- AI摘要回答（文字，≤1000字符）
- 参考来源列表（新闻标题+链接，≤5条）

**业务流程**：
1. 用户输入问题
2. 系统提取问题关键词
3. 向量检索匹配相关新闻（Top 10）
4. 构建Prompt，包含问题和相关新闻
5. 调用LLM生成摘要
6. 展示回答 + 来源列表

**关键规则**：
- 如无匹配结果，提示"暂无相关内容"
- 单次会话最多10轮对话
- 回答引用新闻需包含原文链接

## 7. 技术方案概要

### 7.1 技术架构

```
用户输入 → 关键词提取 → 向量检索(FAISS) → LLM调用 → 回答生成 → 展示
```

### 7.2 数据存储

| 数据 | 存储方案 | 说明 |
|------|----------|------|
| 新闻向量 | FAISS / Vector DB | 预计算，每周更新 |
| 问答历史 | LocalStorage | 浏览器本地存储 |
| 用户反馈 | MySQL | 后续接入 |

### 7.3 LLM 选型

| 方案 | API | 成本 |
|------|-----|------|
| MiniMax | MiniMax API | 中等 |
| OpenAI | OpenAI API | 较高 |

## 9. 风险与应对

| 风险 | 影响 | 应对方案 |
|------|------|----------|
| LLM响应慢 | 体验差 | 增加loading状态 + 超时提示 |
